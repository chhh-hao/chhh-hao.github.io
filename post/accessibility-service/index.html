Android四大组件

活动（Activity）、服务（Service）、广播接收器（Broadcast Receiver）、内容提供器（Content Provider）

  1. 活动：

    一个Activity通常是一个窗口，Activity之间通过Intent进行通信。

    Activity有四种状态：运行中、暂停、停止、销毁

  2. 服务

    Service不依赖于用户界面，但并不是运行在独立进程中，而是依赖于创建服务所在的应用程序进程。

    startService() 启动的服务，需要调用stopSelf() 或 stopService() 停止；bindService() 启动的服务调用者退出，服务也会随之终止。

  3. 广播接收器

    广播接收器可以对外部事件做出响应。

  4. 内容提供器

    内容提供器可以把指定的数据集提供给其他应用程序，实现数据共享。

Accessibility Service

基本步骤：声明 → 配置 → 重写方法

声明

在清单的 application 元素中添加一个 service 元素。

intent-filter是无障碍服务intent过滤器。设置permission 为 BIND_ACCESSIBILITY_SERVICE权限，保证只有系统可以绑定。

  <application>
    <service android:name=".MyAccessibilityService"
        android:permission="android.permission.BIND_ACCESSIBILITY_SERVICE"
        android:label="@string/accessibility_service_label">
      <intent-filter>
        <action android:name="android.accessibilityservice.AccessibilityService" />
      </intent-filter>
    </service>
  </application>

配置

在清单中添加一个引用配置文件的 <meta-data> 元素，。

<service android:name=".MyAccessibilityService">
  ...
  <meta-data
    android:name="android.accessibilityservice"
    android:resource="@xml/accessibility_service_config" />
</service>

需要新建一个xml文件（即meta-data中的resource指定的文件res/xml/accessibility_service_config.xml），设置该服务能够处理的无障碍事件类型以及有关该服务的其他信息。

<accessibility-service
     android:accessibilityEventTypes="typeAllMask"
     android:packageNames="com.example.android.myFirstApp"
     android:accessibilityFeedbackType="feedbackSpoken"
     android:notificationTimeout="100"
     android:settingsActivity="com.example.android.apis.accessibility.TestBackActivity"
     android:canRetrieveWindowContent="true"
/>

上述代码中提到的可设置的xml属性：

accessibilityEventTypes : 服务接收的事件类型

常量 描述
typeAllMask 所有类型的事件
typeAnnouncement 应用程序发布公告的事件
typeAssistReadingContext 辅助用户读取屏幕的事件
typeContextClicked 点击内容的事件
typeGestureDetectionEnd 检测到手势的事件
typeNotificationStateChanged 显示通知的事件
typeViewAccessibilityFocused 获得可访问性焦点的事件
typeViewClicked 点击View的事件
……

accessibilityFeedbackType : 设置反馈给用户的方式

常量 描述
feedbackAllMask 所有类型的反馈
feedbackAudible 可听的反馈
feedbackGeneric 普通的反馈
feedbackHaptic 触觉反馈
feedbackSpoken 语音反馈
feedbackVisual 视觉反馈

notificationTimeout ：响应时间

canRetrieveWindowContent : 是否能够检测活动窗口内容的属性

重写方法

Accessibility Service方法：

  1. onServiceConnected() ：成功连接到无障碍服务时，调用此方法。
  2. onAccessibilityEvent() ：检测到与无障碍服务指定的事件过滤参数匹配的 AccessibilityEvent 时，会回调此方法。
  3. onInterrupt() ：当系统要中断服务时，会调用此方法。
  4. onUnbind() ：当系统将要关闭无障碍服务时，会调用此方法。

在MainActivity中开启服务

写一个if判断 accessibility service 是否开启，并在if下写入如下代码

try{
	startActivity(new Intent(Settings.ACTION_ACCESSIBILITY_SETTINGS)); // 打开设置中的开启无障碍服务界面
}catch(Exception e)
{
	startActivity(new Intent(Settings.ACTION_SETTINGS));
	e.printStackTrace();
}

判断accessibility service是否开启的代码如下：

public static boolean isAccessibilitySettingsOn(Context mContext, Class<? extends AccessibilityService> clazz) 
{
    int accessibilityEnabled = 0;
    final String service = mContext.getPackageName() + "/" + clazz.getCanonicalName();
    try {
        accessibilityEnabled = Settings.Secure.getInt(mContext.getApplicationContext().getContentResolver(),
                Settings.Secure.ACCESSIBILITY_ENABLED);
    } catch (Settings.SettingNotFoundException e) {
        e.printStackTrace();
    }
    TextUtils.SimpleStringSplitter mStringColonSplitter = new TextUtils.SimpleStringSplitter(':');
    if (accessibilityEnabled == 1)
    {
        String settingValue=Settings.Secure.getString(mContext.getApplicationContext().getContentResolver(),
				Settings.Secure.ENABLED_ACCESSIBILITY_SERVICES);
        if (settingValue != null)
        {
            mStringColonSplitter.setString(settingValue);
            while (mStringColonSplitter.hasNext())
            {
                String accessibilityService = mStringColonSplitter.next();
                if (accessibilityService.equalsIgnoreCase(service))
                    return true;
            }
        }
    }
    return false;
}

应用

通过上述步骤，我们已经创建了自己的Accessibility Service，只需要在上述框架的基础上补充代码即可实现想要的功能。

比如我们设置的接收事件为点击事件，那么我们可以在 onAccessibilityEvent() 方法中判断 event.getText() 是否等于我们想要点击的Button或TextView等。

或者我们可以通过 getRootInActiveWindow() 方法获得该界面的AccessibilityNodeInfo根节点，然后通过遍历节点，得到界面中所有控件的信息。

遍历节点的代码如下：

public void onAccessibilityEvent(AccessibilityEvent event) {
    // 其他操作
    AccessibilityNodeInfo currentNode = getRootInActiveWindow();
	if(currentNode == null)
		Log.i(TAG, "Root node info is null");
	else
		recycle(currentNode);
}

public void recycle(AccessibilityNodeInfo info)
{
    if(info.getChildCount() == 0)
        // 对子节点执行的操作
    else
    {
        for(int i = 0; i < info.getChildCount(); i++)
            if(info.getChild(i) != null)
                recycle(info.getChild(i));
    }
}
