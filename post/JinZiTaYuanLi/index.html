金字塔原理读书笔记

第一篇 写作的逻辑

文章条理不清大多数时候与文章的结构有关。对受众来说，最容易理解的顺序是，先了解主要的抽象的思想，然后了解次要的、为主要思想提供支持的思想。

在金字塔结构中，思想之间的联系方式可以是纵向的，也可以是横向的。在纵向思想联系方式中，每一个层次都是对下一个层次的概括。在横向思想联系方式中，多个思想共同属于同一逻辑范畴，并且按照逻辑顺序组织。

金字塔结构组织的思想为：从金字塔的顶部开始，沿各个分支向下展开。首先表达主要思想，使读者产生疑问。然后主要思想的下一层思想对疑问做出回答。通过疑问回答式对话，受众可以了解全部思想。

第一章 为什么要用金字塔结构

基本上，大脑会认为一同发生的任何事物之间都存在某种关系，并且会将这些事情按某种逻辑模式组织起来。当大脑发现其需要处理的项目数量增加到4个或5个以上时，就会开始将其归类到不同的逻辑范畴中，以便于记忆。将思想或概念分组时，应该找出其中的逻辑关系。

控制表达思想的顺序是做到清晰写作的惟一最重要的行为，而最清楚的顺序就是先提出总结性思想，然后再提出被总结的具体思想。条理清楚的书面文书应当具有金字塔结构，并且不断“自上而下”地向读者传递信息。

第二章 金字塔内部的结构

纵向：疑问/回答式对话——确定某一层次上的思想组必须表达哪种信息，并且回答上一层次提出的疑问。

横向：演绎或归纳逻辑——组织在一起的思想应该符合逻辑地表达了某种信息。

序言结构：情景、冲突、疑问、回答——开头应向读者说明“情境”的时间和地点，在这一“情境”中应当发生了某件事情，称为“冲突”，由冲突引发“疑问”，而这个“疑问”正是文章要“回答”的。

第三章 如何构建金字塔结构

✦ 自上而下法

  1. 画出主题方框
    准备讨论的问题 → 有关该主题的疑问 → 回答疑问 → 列出背景 → 列出冲突 → 检查疑问答案是否成立 → 上述答案引发新的疑问 → 使用演绎或归纳法回答新疑问 → 重复上述步骤
  2. 设想主要疑问
  3. 写出对该疑问的回答
  4. 说明背景
    把要讨论的主题与背景相结合，作出第一个不会引起争议的表述。
  5. 指出冲突
    考虑背景中有哪些能使读者产生疑问的冲突。
  6. 检查主要疑问和答案
    保证背景中的冲突能够让读者直接提出主要疑问。

✦ 自下而上法
无法确定要讨论的主题，无法构建金字塔结构的顶部时，而已使用自下而上的方法组织思想。

  1. 列出你想表达的所有思想要点
  2. 找出各要点之间的逻辑关系
  3. 得出结论

✦ 注意事项

  1. 先搭结构，先使用自上而下的方法
  2. 序言先写背景，将背景作为序言的起点
  3. 序言要多花时间思考
  4. 将历史背景放在序言中
  5. 序言仅涉及保证真实性的内容
  6. 关键语句选择归纳推理法，而非演绎论证法。

第四章 序言部分的具体写法

✦ 讲故事形式：情境——冲突——解决方案（各部分的顺序可以有变化，以创造文章的不同风格）

关键句要点必须表达文章的思想
反例：
最好永远不要用“背景”或“介绍”作为某一章节的标题，因为这种标题内所含信息同其他标题所含信息肯定不在同一抽象层次上，或者说抽象的程度不同。
列出主语而不列出完整的思想会存在一种危险，即关于该主语的思想无法形成明确的归纳或演绎逻辑论证关系。不要将某个名词范畴作为关键句的要点，而应当是完整的思想性。

✦ 序言常见模式

  1. 发号施令式（针对“我们应该做什么”或“我们应该如何做”之类问句）
    S=我们打算做X
    C=需要你们做Y
    Q=我们如何做Y
    S=你做X
    C=必须以Y方式进行
    Q=什么是Y方式

  2. 寻求支持式（针对“我们是否应该这样做”之类问句）
    S= 我们遇到了一个问题
    C=我们的解决方案需要经费
    Q=（我应该同意吗）此类疑问通常是隐藏的，而且答案是肯定的。

  3. 解释方法式（针对“我们应该如何做”之类问句）提供咨询时
    S=必须做Y
    C=还未做好做Y的准备
    Q=如何做好准备
    S=你们目前的系统是Y
    C=该系统不能正常工作
    Q=如何改进，使其正常工作？

  4. 比较选择式（针对“我们应该做什么”之类问句）
    S=我们希望做Y
    C=我们有一些做Y的不同方案
    Q=哪一种方案最为合理
