《计算机组成原理》读书笔记

计算机系统体系结构

计算机系统——中央处理单元 + 存储器 + 其他子系统

1.  中央处理单元：读取并执行程序
2.  存储器：保存程序和数据
3.  其他子系统：将芯片转换为实用系统

处理器、微处理器

1. 处理器：实际执行程序的部分
2. 微处理器：单个硅片上实现的CPU

如果不能高效地进行数据传输，仅仅提高CPU的性能毫无意义。

存储器

1. Cache：存放常用数据的高速、专用存储器
2. 主存：存放大量工作数据
3. 辅存：磁盘和CD-ROM，存放海量数据，价格便宜

总线

一些计算机系统使用总线**扩展接口**和**桥接技术**，能够在不同类型的总线间进行数据交换。

体系结构和组成

  1. 体系结构：计算机的抽象指令集体系结构。

    指令集体系结构包括：数据类型、用来保存临时结果的寄存器、指令的类型和格式、寻址方式

    寄存器：存放一个单位的数据或字数据的存储单元，通常用它所保存的数据的位数来描述，如8位、16位、32位、64位。

    寄存器和存储器的实际差别在于，寄存器位于CPU内，它的访问速度远远快于CPU外的存储器。

  2. 组成：计算机的实际硬件实现。

    可以说计算机组成是计算机体系结构的实例化。

  3. 影响计算机设计的因素

    20世纪70年代以来，半导体技术一直按照摩尔定律的预测发展，即芯片的集成度每18个月翻一番。也就意味着处理器的性能也可以说每18个月翻一番。

计算机发展

  1. 机械计算机
    17世纪法国数学家、德国数学家发明了不同的机械计算装置，但是都不能称作现代意义上的计算机，因为它们都是不可编程的。
    可编程：产生于工业革命时期，出于工业化控制的需要。（提花织机）
    19世纪中期，巴贝奇设计处法分析机。分析机是一个完整的机械装置，拥有完成计算的计算单元、输入和输出数据的手段、以及一个基于穿孔木片的存储器来保存数据和程序。
    英国数学家阿达在19世纪中期重写了包括用分析机计算伯努利数的一系列注释，被认为是编写计算机程序的第一人。
  2. 机电式计算机
    机电：有活动件但却由电控制的零件。
    19世纪末期楚泽制造出自己设计的计算机，被认为是第一台可编程计算机。他也设计了世界上第一种程序设计语言。
  3. 早期电子计算机
    真空管放大器取代继电器后，高速、自动的计算成为可能。
    在20世纪前期，出现了第一台电子计算机、巨像计算机、ENIAC（真空管计算机）、EDVAC（可存储程序的真空管计算机）、以及第一台可操作的存储程序计算机。
  4. 微机和PC革命
    集成电路技术的出现为微型计算机铺平了道路。
    20世纪70年代，Intel和摩托罗拉都发布了8位微处理器。
    基于Intel的技术和微软的操作系统软件，IBM在20世纪80年代推出了个人电脑。
  5. 摩尔定律和进步的历程
    摩尔定律：集成电路的集成度每两年会翻一番。
    计算机指令集体恤结构没有特别显著的改进——Pentium系列处理器的指令与Intel80386区别不大，但微体系结构领域产生了惊人进展。RISC革命提出指令流水线的概念，将处理器变成一个用自动生产线执行指令的工厂。
    超标量处理和乱序执行的出现是计算机组成的另一个重要变化。超标量处理包括从存储器中读出几条指令且并行执行这些指令。乱序执行指以不同于程序中的顺序执行指令，以避免等待某条指令的执行，从而加快指令的执行速度。
  6. 存储技术的发展
    最早的存储设备是一个覆盖着电容的旋转磁鼓，是真正的动态存储。
    第一个快速数据存储设备是威廉姆斯管。
    20世纪中期出现了铁氧体磁芯存储器，70年代发明了半导体动态存储器用作磁芯存储器的替代品，现在已经成为数据存储的标准手段。
    磁盘一直用来存放程序和数据。
  7. 普适计算
    普适计算的含义是计算无处不在，主要特征为普适性。
  8. 多媒体计算机
    多媒体处理能力是现代计算机的一个重要特征。

存储程序计算机

算法、程序和伪码

1. 算法：用来完成某个功能的长度有限的明确定义的指令序列。 
2. 程序：实现一个算法的一组计算机指令
3. 伪码：用特定的类程序设计语言描述的算法

存储程序计算机会将指令、变量、常量全部保存着同一个存储器内。存储器中的每个位置要么保存指令，要么保存数据元素。

存储系统的组成：处理器发送地址、控制信号给存储器。存储器读取数据时，把数据放在数据总线上供CPU读取；写数据时，放在数据总线上的数据被写入存储器。信息进入或离开存储器的位置叫端口。

真正的计算机会使用存储系统层次。这些层次包括保存频繁被访问数据的速度快的Cache、主存，以及速度慢的**辅存。**辅存中大量数据会保存着磁盘、光盘、DVD中，直到使用时才被调入主存。

寄存器传输语言RTL：[15] = 5：地址为15的存储单元的值为5

存储程序的概念

存储程序计算机的基本操作：

程序计数器指向存储器中的第一条指令
	REPEAT
		从程序计数器所指的存储单元中读出指令
		修改程序计数器，使其指向下一条指令
		将从存储器中取出的指令解码
		执行指令
			IF 指令需要数据
				THEN 从存储器中读这个数
			END_IF
			完成指令定义的操作
			IF 指令要将数据写回存储器
				THEN 将数据写回存储器
			END_IF
		END
	FOREVER
END

上述伪码表明，从存储器中取出每条指令都需要进行一次访存操作***。在这台计算机上，执行一条指令需要至少两次访存操作：一次读取指令，一次取出指令需要的数据，或将产生或修改的数据写回存储器。有时候称存储程序计算机上按照**读取/执行***周期的两阶段模式工作。

冯诺依曼瓶颈：CPU和存储器之间的通路上存储程序计算机的制约因素之一。

存储程序计算机执行指令的格式

  1. 一般性指令

    Operation Address1, Address2, Address3

    Operation: 要执行的指令的动作

    Address1: 数据的目的地址

    Address2, Address3: 源操作数地址

  2. 两地址指令

    Operation destination, source

    Operation: 要执行的指令

    destination: 既是源操作数，也是目的操作数

    source: 源操作数

    实际的计算机中，一般都不会允许同一条指令中使用两个存储地址，绝大多数计算机都规定一个地址是存储器地址，另一个是寄存器地址。

  3. 单地址指令

    Operation address

    另一个操作数来自CPU中的叫做累加器的寄存器。操作数需要先加载到累加器中，然后再对执行操作，最后把累加器中的结果保存着第三个操作数中。

    寻址方式：计算机指令集的一个重要特征，它是确定操作数位置的方法，可以直接或间接给出操作数位置。有效地址是一种更一般的寻址形式，它代表了生成操作数地址的所有不同方式，避免指定某个特定的寻址方式。

计算机分类

  1. 存储器-存储器型

    一条指令能够从存储器中读出源操作数，对数据完成某个操作，并将结果保存在存储器中，这台计算机就叫存储器-存储器型。

  2. 寄存器-存储器型

    使用Intel IA32 CPU的计算机具有寄存器-存储器型体系结构，能够处理两个数据，一个在存储器中，一个在寄存器中。

  3. 寄存器-寄存器型

    某些使用了ARM和MIPS CPU的计算机智能对寄存器中的内容进行操作。这些计算机必须通过LOAD指令将数据读入寄存器，并使用STORE指令将数据从寄存器送回存储器。也叫load/store型计算机。
